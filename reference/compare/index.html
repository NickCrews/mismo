    Comparing API

    Once records are blocked together into pairs, we actually can do pairwise comparisons on them.

    High-level

    mismo.compare.compare(t: it.Table, *comparers: Any) -> it.Table

    Apply the supplied comparers in order to the table.

    PARAMETER DESCRIPTION
    t

    The table to compare.

    TYPE: Table

    comparers

    Each one can be:

    • an Ibis Expression
    • a Deferred
    • a Mapping[str, ...] of column names to expressions
    • a Table, in which case it is returned as is
    • a callable that takes a table and returns one of the above

    TYPE: Any DEFAULT: ()

    RETURNS DESCRIPTION
    Table

    A mutated table with the specified comparers applied.

    Level-Based Comparisons

    Bin record pairs into discrete levels, based on levels of agreement.

    Each LevelComparer represents a dimension, such as name, location, price, date, etc. Each one contains many AgreementLevels, each of which is a level of aggreement, such as exact, misspelling, within_1_km, etc.

    mismo.compare.LevelComparer

    Assigns a level of similarity to record pairs based on one dimension, e.g. name

    This acts like an ordered, dict-like collection of AgreementLevels. You can access the levels by index or by name, or iterate over them. The last level is always an else level, which matches all record pairs if none of the previous levels matched.

    name: str property

    The name of the comparer, eg "date", "address", "latlon", "price".

    __call__(pairs: it.Table) -> it.StringColumn

    Label each record pair with the level that it matches.

    Go through the levels in order. If a record pair matches a level, label it. If none of the levels match a pair, it labeled as "else".

    PARAMETER DESCRIPTION
    pairs

    A table of record pairs.

    TYPE: Table

    RETURNS DESCRIPTION
    labels

    The labels for each record pair.

    TYPE: StringColumn

    __getitem__(name_or_index)

    Get a level by name or index.

    __init__(name: str, levels: Iterable[AgreementLevel | tuple[str, bool | Deferred | Callable[[it.Table], it.BooleanValue]]])

    Create a LevelComparer.

    PARAMETER DESCRIPTION
    name

    The name of the comparer, eg "date", "address", "latlon", "price".

    TYPE: str

    levels

    The levels of agreement. Can be either actual AgreementLevel objects, or tuples of (name, condition) that will be converted into AgreementLevels.

    You may include an else level as a final level that matches everything, or it will be added automatically if you don't include one.

    TYPE: Iterable[AgreementLevel | tuple[str, bool | Deferred | Callable[[Table], BooleanValue]]]

    __iter__() -> Iterator[AgreementLevel]

    Iterate over the levels, including the ELSE level.

    __len__() -> int

    The number of levels, including the ELSE level.

    mismo.compare.AgreementLevel

    Bases: NamedTuple

    A level of agreement such as exact, phonetic, or within_1_day.

    A AgreementLevel is a named condition that determines whether a record pair matches that level.

    condition: bool | Deferred | Callable[[it.Table], it.BooleanValue] instance-attribute

    A condition that determines whether a record pair matches this level.

    Note that if this AgreementLevel is used in conjunction with other AgreementLevels, the condition is implcitly dependent on the previous levels. For example, if the previous level's condition is _.name_l.upper() == _.name_r.upper(), then if this level's condition is _.name_l == _.name_r, this level is useless, and will match no records pairs, because they all would have been matched by the previous level.

    Examples:

    • _.name_l == _.name_r
    • lambda t: (t.cost_l - t.cost_r).abs() / t.cost_l < 0.1
    • True

    name: str instance-attribute

    The name of the level. Should be short and unique within a LevelComparer.

    Examples:

    • "exact"
    • "misspelling"
    • "phonetic"
    • "within_1_day"
    • "within_1_km"
    • "within_10_percent"

    is_match(pairs: it.Table) -> it.BooleanColumn

    Return a boolean column for if the record pair matches this level.

    Plotting

    mismo.compare.compared_dashboard(compared: Table, comparisons: Iterable[LevelComparer], weights: Weights | None = None, *, width: int = 500) -> ipywidgets.VBox

    A dashboard for debugging compared record pairs.

    Used to see which comparison levels are common, which are rare, and which Comparisons are related to each other. For example, exact matches should appear together across all Comparisons, this probably represents true matches.

    PARAMETER DESCRIPTION
    compared

    The result of a comparison.

    TYPE: Table

    comparisons

    The Comparisons used to compare the records.

    TYPE: Iterable[LevelComparer]

    weights

    The Weights used to score the comparison, by default None If provided, the chart will be colored by the odds found from the Weights.

    TYPE: Weights | None DEFAULT: None

    width

    The width of the chart, by default 500

    TYPE: int DEFAULT: 500

    RETURNS DESCRIPTION
    VBox

    The dashboard.
